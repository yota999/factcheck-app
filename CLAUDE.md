# 台本生成システム

## このプロジェクトについて
- Streamlit製のWebアプリ（YouTube・リール動画の台本生成）
- GitHub: yota999/factcheck-app（masterブランチ）
- 本番URL: factcheck-app-appppdkmaxjagghw9gtgak6p.streamlit.app
- Streamlit Cloudに自動デプロイ（push後1〜2分で反映）

## ファイル構成
- `script_crew.py` — 台本生成のAIエージェント処理（メインロジック）
- `memory_manager.py` — 学習データ・履歴管理（Supabase）
- `pages/1_台本生成.py` — 台本生成UI（Streamlit）
- `pages/2_生成履歴.py` — 生成履歴UI
- `app.py` — トップページ（ナビゲーション）

## 技術スタック
- Python / Streamlit（wide layout）
- LiteLLM（4AI並列呼び出し）
- Supabase PostgreSQL（学習データ・履歴のクラウドDB）
- ThreadPoolExecutor（4本並列生成）

## 台本生成の仕組み
### 生成フロー
1. Step 0: YouTube台本 or リール台本を選択 → 即Step 1へ
2. Step 1: 素材テキストを入力（ブログ・メモ・箇条書きなど何でもOK）
3. Step 2: 4つのAIが並列で台本を生成 → タブで読み比べて1本を選択
4. Step 3: 選択した台本を部分修正・ブラッシュアップ → 完成

### 使用モデル（4本並列）
- Claude Sonnet 4.6（anthropic）
- GPT-4o 2024-11-20（openai）
- Gemini 2.5 Flash（google）
- Grok 3（xai）

### ペルソナ設定
- コーチ名：町田耀大
- 実績：トレーナー歴15年・6500人以上指導
- ターゲット：35歳以上（主に40〜50代）の女性
- 一人称：僕 / 二人称：あなた・女性

### CTA（固定・自動追加）
- YouTube：`YOUTUBE_CTA`（締め文＋LINE誘導＋チャンネル登録）
- リール：`REEL_CTA_FOOTER`（固定）＋ `_generate_reel_cta_intro()`（台本内容に合わせて動的生成）
- AIはCTAを生成しない。生成後に自動でくっつける

### 学習システム
- 台本を修正するたびに改善ルールを自動抽出・保存（最大20件）
- `memory/history.json`（ローカル保存・gitignore）
- `get_edit_improvements()` で次回生成時に反映

## 主要な定数（script_crew.py）
- `YOUTUBE_PERSONA` — YouTubeペルソナ＋入力形式
- `YOUTUBE_STRUCTURE` — YouTube台本の7セクション構造ルール
- `YOUTUBE_CTA` — YouTube用固定CTA全文
- `REEL_PERSONA` — リールペルソナ＋入力形式
- `REEL_STRUCTURE` — リール台本の6セクション構造ルール
- `REEL_CTA_FOOTER` — リール用固定CTA末尾

## 文字数目標
- YouTube台本：5000文字以上（CTAは別途自動追加）
- リール台本：850〜950文字（CTAは別途自動追加）
- 不足時は自動展開パス（4700文字未満で2回目の生成をかける）

## YouTube台本の科学的表現品質ルール（YouTube専用）
やること：
- 専門用語は一段階深い構造レベルで使う（レジスタントスターチ→アミロースの再結晶化など）
- メカニズムの結果は「鏡を見たときの変化（部位・見た目）」で表現する
- 科学的メリット＋日常生活のメリットを複数重ねる
- 難読漢字には読み仮名をつける（例：再結晶化（さいけっしょうか））
やらないこと：
- 「レジスタントスターチ」「カプサイシン」など知られすぎた用語をそのまま使わない
- 「血糖値が上がる」で終わらせない（必ず脂肪・部位の変化に変換）
- 表面的な現象説明だけで終わらせない（構造・メカニズムまで一段階深く）
- YOUTUBE_STRUCTUREの【科学的表現の品質ルール】に記載済み

## 専門用語の言い換えルール（YouTube・リール共通）
- **専門用語の直後に必ず「つまり〜」で一般女性向けの言い換えを入れること**
- 言い換えの方向性：「自分の体に何が起きるか（脂肪・見た目・体感）」で表現する
- 例：「血糖値が上がる」→「脂肪がつきやすくなる」
- 例：「血糖値が抑制される」→「脂肪がつきにくくなる」
- 例：「インスリンが過剰分泌」→「食べたものが脂肪に変わりやすくなる」
- YOUTUBE_STRUCTURE・REEL_STRUCTURE 両方の【専門用語の言い換えルール】に記載済み

## YouTube台本のステップ解説品質ルール
- **複数ステップを生成するとき、食材・行動・メカニズムがすべて異なること**
- NG例：「豆乳を飲む」＋「豆乳から先に飲む」→ 同じ食材の使い方違いは重複とみなす
- OK例：「豆乳を飲む」＋「酢を加える」→ 別の食材・別のメカニズム
- 各ステップを一言要約して重複がないか自己チェックしてから出力させる
- YOUTUBE_STRUCTURE の【ステップ間の重複・類似禁止ルール】に記載済み

## YouTube台本の必須記載ルール
- **「町田式オンラインダイエットの運動量は1日2〜3分のお家での筋トレのみ」を必ずどこかに入れること**
- 適用範囲：YouTube長尺動画台本のみ（リール台本は対象外）
- 場所：ステップ解説・まとめ・締め文のいずれかに自然な形で盛り込む
- YOUTUBE_STRUCTURE の【必須記載事項】セクションに記載済み

## 出力形式のルール
- **Markdown記号（###・##・#・**・*・---）は台本に絶対に含めないこと**
- プロンプトで明示的に禁止済み（REEL_STRUCTURE・YOUTUBE_STRUCTURE 両方の文体ルール）
- 後処理（`_clean_output()`）でも全4AIの出力から自動除去する（二重保護）

## 文字数カウントのルール
- **文字数カウントは必ず Python の `len()` を使うこと**（`wc -m` は使わない）
- 理由：日本語は1文字が複数バイトのため `wc -m` とカウント結果がズレる場合がある。`len()` はUnicode文字単位で正確に数える
- 適用範囲：リール台本・YouTube台本どちらも同じルール
- 現在の実装：`script_crew.py` / `pages/1_台本生成.py` ともに `len()` で統一済み
